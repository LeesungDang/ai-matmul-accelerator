## Design Comparison

| Design | Style | Approximate Behavior | Tradeoff |
|---|---|---|---|
| matmul_4x4 | Combinational | Computes all outputs in one combinational block | Fast in cycle count, more parallel hardware, longer combinational path |
| matmul_4x4_seq | Sequential FSM | Computes one multiply-accumulate step per cycle | Fewer compute resources, more cycles, clearer accelerator control |

## Verification Notes

Randomized testing uses signed INT8 input values and wider expected outputs generated by Python/NumPy.

This caught signed-arithmetic issues that fixed positive-only tests did not catch.

Key lesson:

INT8 inputs require wider signed accumulation for matrix multiplication.
